一、主要针对Oracle转MySql，包括：
1、完全准确：
NVL -> IFNULL
SYSDATE -> NOW()
SYSTIMESTAMP  -> NOW()
SYS_GUID -> UUID()
NUMBER -> DECIMAL
TO_CHAR -> DATE_FORMAT：日期分隔符为-或/，时间分隔为:
TO_DATE -> DATE_FORMAT：日期分隔符为-或/，时间分隔为:
TO_DATE + 1 -> DATE_ADD
DECODE -> CASE WHEN：嵌套的DECODE也支持。
2、可能会有误差：
|| -> CONCAT：无法使用正则完全匹配边界。

二、无法解决的：
ROWNUM = 1 -> LIMIT 1：部分子查询里会报错
自定义函数：无法转换
