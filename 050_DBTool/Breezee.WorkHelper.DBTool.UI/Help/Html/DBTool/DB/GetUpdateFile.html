获取变化的文件

一、功能概述

很多时候，我们都是使用Git来做源码管理，但Git存在文件被他人覆盖的风险。为了代码安全，我们应该时常备份自己修改的文件（再放SVN做源码管理），那么可以使用本功能。 在提交前，获取新增或修改的文件是最准的。但提交之后，再使用本功能获取，还是存在获取到帮别人提交的部分代码文件，这个问题暂时未能解决， 不过自己修改并提交的文件是肯定包含在内的！多备份他人的文件只是多占一点空间而已，影响不大！！

三、操作步骤

1、【读取目录】：输入或选择要读取源码的目录。我们最好将所有项目放在一个目录下，方便一次备份所有项目的文件！
2、【生成目录】：备份文件的目录，我们可以将该目录作为SVN目录，当备份完成后，再提这些文件提交到SVN上做二次源码管理。
3、【修改时间】：我们可以选择开始和结束时间。
保存结束时间：复选框，当选中时，获取文件后，会记录获取时的结束时间，并记录起来，在下次获取时可以从该时间开始获取。
从上次保存的结束时间开始：复选框，当选中时，开始时间就会取上次获取时的结束时间。
获取时结束时间为当前时间：复选框，当选中时，那么结束时间是点【获取文件】时的时间，不点【获取文件】时结束时间是未定的。
4、【源目录类型】：可选择【含git源码的目录】或【普通目录】
【普通目录】：一般的文件目录，会根据文件的最后修改时间来确定文件清单！
【含git源码的目录】：针对纳入git源码管理的目录，还有以下选项：
含正在修改：包含目录正在修改的已纳入源码的文件。
含将新增：包括将要新增的文件。
含之前提交：包括指定时间内，提交人为Email或用户名的人提交的文件。 Email：提交人的Email地址。 用户名：提交人的用户名。 5、排除目录或文件选项：
排队目录名：排除文本框内输入的所有子目录。
排除绝对目录：排除文本框内输入的所有全路径的子目录。
排除后缀：排除文本框内输入的指定后缀的文件。
排除文件：排除文本框内输入的指定文件名的文件。
排除绝对文件名：排除文本框内输入的指定全路径文件名的文件。
4、点击【获取文件】：将会异步获取文件。
获取成功后，会在【生成的字符】中显示本次获取到的文件清单。

四、逻辑与业务规则

注意点：支持逗号（中英文）、分号（中英文）、冒号（中文）、竖线（英文）分隔的多个排除项配置！另外：会过滤以.开头的文件夹。
