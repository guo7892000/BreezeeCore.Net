字段默认值与条件配置

一、功能概述

本功能维护的数据是提供给【表增删改查SQL生成】功能在勾中【使用全局配置】时使用。 使用场景包括：统一设置表中通用字段的新增和修改SQL语句中的默认值；条件中一般会添加的通用查询条件。 当然，该配置是可选的；针对每个表的增删改查SQL语句生成，我们都可在该功能中具体设置而不使用该配置！！

三、操作步骤

主页面： 【查询】：可以查询当前已维护好的所有默认值设置信息。
【新增】：弹出【默认值设置明细】窗体。
【编辑】：需要选择一行数据，然后双击或点【编辑】按钮，则会弹出【默认值设置明细】窗体。
编辑页面： 1、填写默认值设置相关信息：
字段名：必填
SqlServer默认值：针对SqlServer数据库类型的默认值。
Oracle默认值：针对Oracle数据库类型的默认值。
MySql默认值：针对MySql数据库类型的默认值。
PostgreSql默认值：针对PostgreSql数据库类型的默认值。
SQLite默认值：针对QLite数据库类型的默认值。
默认值适用语句：包括【新增语句】、【修改语句】
字段条件类型：包括【修改条件】、【查询条件】、【删除条件】。
状态：包括【启用】复选框，选中即为启用，否则为禁用。
2、点击【保存】：
保存成功后，会提示相关信息。

四、逻辑与业务规则

【默认值】：针对五种常见数据库类型的独立设置。
【默认值适用语句】：只有勾选了【新增语句】、【修改语句】，才会在这两种语句类型和数据库类型上使用相应的默认值。
【字段条件类型】：只有勾选了这三种条件，才会在这三种语句类型上加上该字段作为条件。
