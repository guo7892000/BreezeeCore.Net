SQL跨库转换

一、功能概述

将某个数据库类型的SQL，转换为符合另一种数据库类型的SQL。
注：因为SQL语句本身非常复杂，无法考虑所有情况，所有这里只针对一些常见的函数做转换，针对指定的新旧列关系替换旧字段名。 只是减少一些替换工作量；转换结果仅供参考，最终还是需要人工再次确认修改！

三、操作步骤

有两种方式选择，包括：
1、自定义：
通过自定义的新旧表关系，或新旧列关系。操作步骤如下：
1.1、【新旧列关系获取方式】：选择【自定义】
1.2、关系录入：
【新旧表】页签：如果有该页签内容，并且表存在于SQL，那么转换会使用这里的新旧表关系替换。
【新旧列】页签：如果有该页签内容，并且表和列存在于SQL，那么转换会使用这里的新旧列关系替换表名和列名。
1.3、输入【要转换的SQL】：输入SQL。
1.4、点击【生成】：转换后的SQL将显示右侧的生成结果中。

2、读取数据库：
针对新旧数据库中新旧列的备注中文名一致时才转换（匹配率低，不建议）；此方式下，必须选择源和目标数据连接。操作步骤如下：
2.1、【新旧列关系获取方式】：选择【读取数据库】
2.2、数据库连接选择：
【源数据库】：SQL中的旧列名来源于该连接。
【目标数据库】：SQL中的新列名来源于该连接。
2.3、点击【连接加载所有】：将分别连接源数据库、目标数据库，并同时加载所有表及列信息。
2.4、【新旧表】页签：必填，可以提前维护好新旧表模板，再选择。通过这个关系得到旧表字段的中文名，再从新表中找出该名称的列名来替换
【新旧列】页签：可选，如果有该页签内容，并且表和列存在于SQL，那么转换会优先使用这里的新旧列关系替换表名和列名。
2.5、输入【要转换的SQL】：输入SQL。
2.6、点击【生成】：转换后的SQL将显示右侧的生成结果中。

四、逻辑与业务规则

【新旧列】关系替换：只能针对【别名.列名】的列进行替换，另外它比读取数据的新旧列关系优先。
如果【新旧表】、【新旧列】两个页签都没内容时，那么转换时只会进行函数替换。
