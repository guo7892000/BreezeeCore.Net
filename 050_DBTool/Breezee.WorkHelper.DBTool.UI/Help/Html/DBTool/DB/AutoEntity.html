实体类生成

一、功能概述

针对数据库中的表、视图，生成实体对象，方便于界面赋值与生成数据库变更SQL。包括【生成实体对象】、【实体模板】、【参数配置】3个页签。

二、界面截图

生成实体对象：操作实体生成

实体模板：根据模板来生成实体

参数配置：模板中的一些替换配置

三、操作步骤

1、选择【连接配置名】：从下拉框中选择。数据来源于【数据库连接配置】功能。
2、点击【连接服务器】：连接成功后会加载左边树，包括表清单。
3、录入【生成的类信息】：保存路径为必填。类名前缀、类名后缀、属性前缀为可选项。
4、展开树，选择子节点：

全部生成：如果选择的是数据库节点，则右击选【全部生成】，可生成多个表的实体类文件。
单表生成：如果选择的是表节点，则右击选【单表生成】，可生成该表的实体类文件。

四、逻辑与业务规则

实体模板： 实体总文件模板：是生成实体的总模板，包含一些固定参数键。
列名子属性模板：是对所有列根据模板及参数配置分别追加构造，最终替换掉【#COLUMN_PROPERTY_VAR#】。
列名子字符模板：是对所有列根据模板分别替换构造，最终替换掉【#COLUMN_STATIC_STR#】。
参数配置：
参数设置：对列元数据进行配置，以支持列名子属性的构造。
参数说明：对模板中以##包围的参数进行说明。
