增删改查SQL生成

一、功能概述

查询数据库，生成某个表的新增、修改、查询、删除语句。

二、界面截图

主界面：操作文件生成

三、操作步骤

1、选择【连接配置名】：要连接哪个数据库。
2、输入【选项配置】相关信息：
【SQL类型】下拉框：必填，包括新增、修改、查询、删除。
【表名】下拉框：必填，可以手工输入；也可以在勾中【获取表清单】复选框后下拉选择。
【获取表清单】复选框：当选中时会把表清单加载到【表名】下拉框中。
【别名】文本框：表示SQL中表的别名，可空。
【参数类型】下拉框：包括【左右#号】、【SQL参数化】、【MyBatis参数】、【MyBatis动态参数】、【自定义】。
【首字母方式】下拉框：包括【无】、【大驼峰式】、【小驼峰式】。
【参数前缀】文本框：针对不同参数类型的处理不同：
左右#号：不可见
SQL参数化：显示名称为“参数前缀”，值为@
MyBatis参数、MyBatis动态参数、自定义：修改显示名称为“列名替代符”，值显示为@
【参数格式】：针对不同参数类型的处理不同：
左右#号、SQL参数化：不可见
MyBatis参数：显示为参数格式，值为#{param.@}
MyBatis动态参数：显示为字典参数前缀，值为param.
自定义：显示为自定义格式，值为空
【使用全局配置】复选框：当选中时，会结合【SQL类型】与【字段默认值与条件配置】功能维护的值，来修改网格中某些列的值。
【备注】复选框：默认选中，表示SQL中包括注释。如不选中，则只有纯SQL。
【忽略默认值】复选框：默认选中，表示SQL中所有列不使用默认值，都需要参数传入。如不选中，则会在SQL中使用默认值。
【换行】复选框：默认选中，表示SQL中每列都会换行，样式比较好看；如不选中，那么整个SQL会只显示为一行。
3、点击【生成】：成功后会提示相应信息。

四、逻辑与业务规则

【使用全局配置】复选框： 如果我们在设置数据库表时，添加了一些通用列，并且希望在SQL语句中增加统一的处理，我们才会使用该配置选项； 否则直接在本界面中定制各项值就好了。
